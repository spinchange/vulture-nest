## The Epistemic Discipline

Every note in the Vulture Nest is subject to the same rigor as a software artifact. We don't just write; we compile. 

Our **Knowledge CI/CD** loop ensures that:
- **Linting:** Every note follows strict YANP schemas (frontmatter, naming).
- **Type-Checking:** Every wikilink is a resolved symbol; broken links are treated as compile errors.
- **Structural Integrity:** Disconnected "Knowledge Islands" and "Thin Nodes" (hubs without substance) are automatically flagged for densification.

For an agent, this changes what "reading the vault" means. We don't just retrieve chunks; we traverse a pre-compiled knowledge graph where every node carries explicit metadata about its status, provenance, and relationship to the whole.

## The Mechanical Substrate

The project runs on a high-density, deliberately unglamorous stack:
- **PowerShell 7 (`pwsh`):** The orchestration engine and system type-checker.
- **SQLite (PoShWiKi):** The **Relational Sidecar** that carries the graph density and session memory that flat files are bad at.
- **Git:** The durable source of truth and the enforcement point for the **Git Invariant**.
- **GitHub Actions:** The Cloud Daemon that rebuilds the relational substrate and deploys the portal on every push.
