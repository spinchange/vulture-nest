## High-Level Organization
The Python documentation is tiered to support different levels of interaction:
- **Tutorial**: Beginner's guide to syntax and idioms.
- **Library Reference**: Detailed index of standard modules (The "Batteries Included" core).
- **Language Reference**: Formal specification of the core syntax and semantics.

## Essential Module Categories

### Built-in & Runtime
- **Built-in Objects**: Native functions, constants, and exceptions available without imports.
- **`sys`**: System-specific parameters and functions; access to the interpreter.
- **`typing`**: Support for type hints and static analysis.

### Data & Text Processing
- **`json`**: Native JSON serialization and deserialization.
- **`re`**: Regular expression engine.
- **`datetime`**: Date and time manipulation.
- **`collections`**: Advanced data structures (`Counter`, `deque`, `defaultdict`).
- **`itertools` / `functools`**: Functional programming primitives and iterator algebra.

### Filesystem & OS
- **`os`**: Portable operating system interfaces.
- **`pathlib`**: Modern, object-oriented filesystem path manipulation.
- **`shutil`**: High-level file operations (copy, archive).
- **`logging`**: Industry-standard logging framework.

### Concurrency & Execution
- **`subprocess`**: Management of external system processes.
- **`threading` / `multiprocessing`**: Parallelism models (Thread-based vs. Process-based).
- **`asyncio`**: Asynchronous programming and event-loop management.

### Networking & Web
- **`urllib`**: Request handling and URL parsing.
- **`http`**: Basic HTTP client/server implementations.
