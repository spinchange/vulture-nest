# Plain+ Design

**Plain+** is a minimalist design system and aesthetic framework specifically engineered to counter the recognizable "fingerprint" of AI-generated user interfaces. It prioritizes semantic HTML, typographic restraint, and structural clarity over decorative elements.

## Core Philosophy: The Anti-AI Aesthetic
The system rejects common LLM-generated design tropes—such as rounded corners (pill buttons), soft gradients, and "startup blue/purple" color ramps—in favor of an aesthetic that feels editorial, human, and considered.

## Foundational Concepts
- **The "Plus" Factor**: The gap between raw HTML and a professional layout, bridged via fluid type scales and disciplined font pairings.
- **Typographic Contrast**: A core rule pairing **Serif fonts** (for body text) with **System Sans-Serif fonts** (for headings).
- **Warm Neutrals**: Replaces cool digital grays with warm-toned inks and surfaces (#1a1a1a on #fdfdfc) to mimic the feel of paper.
- **Single Accent Discipline**: Exactly one accent color is permitted, used sparingly for specific emphasis or interactive hovers.

## The Ten Non-Negotiable Rules
1.  **No Border-Radius**: All elements must remain square.
2.  **No Gradients or Shadows**: Depth is achieved through hierarchy and spacing, not visual effects.
3.  **No "AI Colors"**: Explicit ban on purples, teals, and SaaS blues.
4.  **Whitespace over Containers**: Use space to separate content; avoid unnecessary boxes.
5.  **Serif-Body-First**: Long-form reading must happen in a serif typeface.
6.  **Thin Rules (1px)**: Use only 1px rules for separation if whitespace alone is insufficient.
7.  **Fluid Scale**: Use modular scales (e.g., 1.25 or 1.333) for all sizing.
8.  **No Zebra Stripes**: Tables use thin rules and alignment, not background shading.
9.  **Vertical Rhythm**: Align all elements to a consistent baseline (e.g., 1.6rem).
10. **Pre-Ship Audit**: Every UI must be audited for "AI tells" before being finalized.

## Application in Vulture Nest
Plain+ is the visual standard for the **Vulture Portal** and all associated dashboards. It ensures that the vault remains a **Content-Forward** artifact where the knowledge remains the focus.
